# 2026-06-30 — Позиционный резолв иерархии сметы (стек по глубине-кода) + детекция аномалий

## Что сделано

Иерархия предков узла сметы (для крошки `embedding_input`) теперь восстанавливается **позиционным
стеком по глубине-кода** (число сегментов), а не усечением точечного кода с подстановкой имени
первого вхождения. Стек смотрит только вверх по документу → forward-ref невозможен, коллизии
дублей кода разводятся естественно, пропущенный код-родитель не теряет цепочку. Параллельно парсер
читает Excel-`outline_level` (openpyxl) **только для детекции** структурных аномалий и агрегата
`outline_overrides`; на резолв outline не влияет (решение «ревью v2»).

Чинит три тихих режима контаминации крошки на кривой нумерации «№ раздела» (многоэтапные сметы):
коллизия дубля кода, «родитель ниже» (`10.2.1.x`), «родителя нет» (газон `11.3.1.1.x`).

Спек: [docs/superpowers/specs/2026-06-30-positional-hierarchy-resolution-design.md](../superpowers/specs/2026-06-30-positional-hierarchy-resolution-design.md)
(врезка «РЕШЕНИЕ РЕВЬЮ v2» — источник правды).
План: [docs/superpowers/plans/2026-06-30-positional-hierarchy-resolution.md](../superpowers/plans/2026-06-30-positional-hierarchy-resolution.md).
Ветка `fix/breadcrumb-ancestor-collision` (база `main`, merge-base `5a2880b`). Поглощает «Кейс D»
([спека](../superpowers/specs/2026-06-30-breadcrumb-ancestor-resolution-design.md) помечена superseded).

## Архитектура

`api → services → domain ← infrastructure` — без изменений направления зависимостей.

- **Домен** ([classification.py](../../backend/app/domain/classification.py)): чистые функции по
  ГЛУБИНЕ-КОДА — `resolve_ancestor_indices(depths)` (стек открытых предков; индексы строго
  возрастают и `< i`), `leaf_flags(depths)` (лист ⟺ следующий не глубже), `canonical_codes(depths)`
  (восстановленное `1/1.1/…` — потребитель в отложенном §5 экспорта), `detect_structural_anomalies`
  (построчные `duplicate_code`/`parent_below`/`parent_missing`/`depth_jump` + агрегат
  `outline_overrides`). `StructuralAnomaly` в [entities.py](../../backend/app/domain/entities.py).
- **Парсер** ([estimate_parser.py](../../backend/app/services/estimate_parser.py)): pandas — значения
  и крошка (глубина из сегментов кода); тонкий проход openpyxl (`wb.worksheets[0]`) — `outline_level`
  только для детекции. Сверка выравнивания pandas↔openpyxl по `source_index+2`: при рассинхроне →
  **warning + `outline_overrides=0`**, ingest НЕ падает (detection-only, крошка код-based не страдает).
- **Сервис матчинга** ([estimate_matching_service.py](../../backend/app/services/estimate_matching_service.py)):
  `_classify_nodes` строит крошку и `is_leaf` позиционно; LLM-контекст предков — из той же цепочки.
  Удалены код-based `_ancestor_codes`/`_parent_of`/`_ancestor_names`/`name_by_code`/`repr_id_by_code`.
- **API** ([schemas.py](../../backend/app/api/schemas.py), [routes/estimates.py](../../backend/app/api/routes/estimates.py)):
  `anomalies` + `outline_overrides` текут в ответ `POST /estimates`.
- **Фронт** ([StructureNotice.tsx](../../frontend/src/components/estimate/StructureNotice.tsx)):
  сворачиваемый блок «Структура сметы» — таблица построчных аномалий + агрегатная строка про outline.

`depth` узла НЕ мутируется (= число сегментов; нужен `reconstruct_nodes`/бенчмарку). §5 (ремонт
нумерации в экспорте) — сознательно вне скоупа; `canonical_codes` в домене есть, потребителя нет.

**`outline_overrides` (ожидание ≈115):** в этой сессии напрямую по golden НЕ снят — файл
`temp/Смета — копия.xlsx` отсутствует в чекауте (приватность), а MinIO-копии заблокированы процессом
сервера на чтение. Путь чтения (`_read_outline`) доказан round-trip-тестом парсера (выровненный
outline `[0,1,2]`→overrides=0; рассинхронный `[0,1,1]`→overrides=1); сам объём ≈115 измерен на этапе
дизайна (§1/§4 спеки). Калибровочная UX-метрика, не гейтит фичу.
