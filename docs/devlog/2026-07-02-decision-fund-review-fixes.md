# 2026-07-02 — Золотой фонд: пакет фиксов пост-мерж ревью (PR #17)

**Ветка:** `feat/decision-fund` (поверх 94f0a7d)
**Спека:** [2026-06-30-decision-fund-design.md §12 (амендменты)](../superpowers/specs/2026-06-30-decision-fund-design.md)
**План:** [2026-07-02-decision-fund-review-fixes.md](../superpowers/plans/2026-07-02-decision-fund-review-fixes.md)

## Что случилось

Глубокое ревью реализации фонда (8 независимых углов поиска + адверсариальная верификация каждого
кандидата) дало 10 подтверждённых находок, включая три ломающих happy path. Все закрыты этим батчем
(TDD: каждый фикс начинался с падающего теста). Два фикса **меняют залоченные решения спеки** —
оформлены амендментами §12 и подтверждены вторым ревью плана.

## Починено

1. **Экспорт терял фонд-строки** (самое серьёзное): `_cell_value` не знал `matched_fund`, а
   «авто-подтверждение» фонда на фронте локальное (`review_status` в БД остаётся `unreviewed`) →
   в .xlsx уезжала пустая «Статья СМР» ровно для строк, закрытых фондом. Заодно продуктовая просьба:
   формат ячейки теперь **«(код) Название»** во всех непустых ветках (амендмент SP3 §5).
2. **500 на тумблере/rebuild**: две одинаковые строки сметы (целевой кейс фонда!) давали дубль
   conflict-ключа в одном bulk-upsert → Postgres `CardinalityViolation`; rebuild умирал ПОСЛЕ
   `clear()`, оставляя фонд пустым. Дедуп в `promote()` + фейк теперь честен по дублям.
3. **Накрутка votes**: повторный toggle-ON той же сметы инкрементил консенсус (`votes+1` без
   условия). Теперь `CASE WHEN source_estimate_id = excluded.source_estimate_id` (§12.5);
   интеграционный тест на реальном Postgres.
4. **Отравленный фонд был неизлечим**: строку `matched_fund` нельзя было ни переопределить в UI
   (некликабельна), ни перематчить (все пути исключали статус). Теперь: строка раскрывается
   (ручной поиск → override; §12.4), `statusLabel` уважает решение оператора (override → «Ручной
   выбор», а не вечное «Из фонда»), а ре-прогон матчинга сбрасывает **unreviewed** фонд-хиты в
   `pending` и переразрешает текущим фондом (§12.3) — контур «unreference + rebuild + re-match»
   стал рабочим.
5. **Молчаливый отскок тумблера**: при `promoted=0` (нет подтверждённых строк в БД) тумблер
   включался и сам выключался без объяснения — теперь toast-подсказка.
6. **Фонд не экономил эмбеддинг**: `_apply_fund` стоял ПОСЛЕ `_embed_nodes` — на повторной смете
   с высоким покрытием все строки оплачивали OpenRouter-эмбеддинги, чьи векторы никогда не
   читались. Стадия переехала ДО эмбеддинга (§12.2), `fetch_unembedded_nodes` исключает
   `matched_fund`; «embed до гейта» и «промах → RAG в том же прогоне» сохранены.
7. **Per-row commits фонд-хитов**: `save_fund_hit` (UPDATE+COMMIT на строку) → bulk
   `save_fund_hits` (один commit на пасс, зеркало `save_node_classifications`).
8. **Мелочи**: `SqlAlchemyDecisionFundRepository` наследует порт-ABC (был единственным адаптером
   без него); перекрёстные комментарии `CRUMB_DERIVATION_VERSION` ↔ `normalize_cache_key` (правка
   нормализации без бампа версии молча остужала бы весь фонд); `toggle_reference` проверяет
   владение лёгким `exists()` вместо `get()` со всеми строками и векторами; CLAUDE.md
   актуализирован (персистентность `decision_fund`, стадия фонда в пайплайне).

## Дополнение (та же дата): гидратация ревью и тумблера

Пользовательский репорт «после перезагрузки ревью не сохраняется» вскрыл: решения оператора
**персистентны** (каждый клик — `PATCH .../review` в Postgres), но при повторном открытии сметы
`initReview` строил решения только из AI-статуса, игнорируя `review_status`/`final_*` — ревью
визуально обнулялось. Починено (TDD):

- `initReview` гидратирует решения через `decisionFromRow` при `review_status != 'unreviewed'`;
  прогресс сразу учитывает решённые строки.
- Тумблер «в фонд»: `Estimate.is_reference` → `EstimateDetailOut` (SQL `_to_entity` + фейк) →
  `getEstimate().isReference` → самогидратация `DoneScreen` на монтировании (guard по `toggleSeq`,
  best-effort). Хвост из TECH_DEBT «гидратация тумблера» погашен.

Замечание UX осталось открытым: на экран «Готово» (с тумблером) можно попасть только через
успешный «Выгрузить Excel» — явной кнопки «Завершить» нет (обсудить отдельно).
